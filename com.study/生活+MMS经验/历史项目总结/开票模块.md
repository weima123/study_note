## 一.业务背景
```
  员工期望少贴票，财务期望员工提供合规的发票，统一开票同时解决了这两个痛点。
```
   
## 二.业务架构
![avatar](file/Invoice.png)
#### 1.业务架构
```
业务系统、开票服务 、滴滴中间件、滴滴供应商
```
```
 1.开票模块从业务系统筛选待开票的订单数据，经由滴滴中间件传输给滴滴供应商去帮用户开具发票。
 2.开票模块根据最终订单的开票结果，对业务系统的数据进行调整。例如：开票失败费用驳回，单据开票通过、单据开票驳回
```

#### 2.开票模块主要的业务功能
```
支持的供应商：
1.神舟
2.滴滴
3.京东
4.公司自营商旅平台：甄选酒店费用开票
```

```
功能角度：
1.手动开票
2.一键开票
3.开票历史查询
4.开票进度查询

```
